¿Qué versiones de JavaScript hay en el mercado?

En el año 1995 nace, por Brendan Eich, el lenguaje Javascript; un lenguaje para interactuar con páginas web. Al año, ECMAScrip un consorcio Europeo que permitieron generar un estándar para que los navegadores lo siguieran (es decir, lenguajes como JS empezaron a seguir estos estándares), lanzando en 1997 los primeros estándares para la primera versión de un lenguaje de scripting llamado ECMAScript. Desde ese momento hasta el día de hoy el grupo TC39 (un grupo de hackers, académicos y distintas personas con conocimiento al respecto) son quienes se encargan de traer nuevas funcionalidades al ECMA.

Las versiones que hay de JavaScript en el mercado son:

  • ES6: entre sus actualizaciones se encuentran ler, const y arrow functions, palabras reservadas para guardar variables.
  • ES7: sus actualizaciones más destacadas son exponentiation operator (una alternativa a Math.pow para poder realizar operaciones más claras) y array includes (para determinar si un elemento se encuentra dentro de un array).
  • ES8: algunas de las actualizaciones son: Object entries , una función que nos permite transformar un objeto en un array con cada una de las propiedades de un array de la forma [key, value]; String padding, un método que permite rellenar la cadena actual con una cadena dada. Las funciones asíncronas permitieron al lenguaje ejecutar código el cual es necesario obtener el resultado previo. Además, viene a solucionar el callback hell, entre otras tantas.
  • ES9: se incorporaron la posibilidad de usar grupos nombrados usando una expresión regular y la posibilidad de usar ciclos de forma asíncrona, esto permitiendo ejecutar código que no bloque el hilo principal de nuestra aplicación.
  • ES10: se mejoró el método flat de los array, ya que permite aplanar los elementos internos de un array permitiendo eliminar array internos.
  • ES11: Promise.allSettled es un método que retorna una promesa después de que todas las promesas dadas fueron completadas resueltas exitosamente o no, con un array de objetos que describe cada una de llamadas.
  • ES12: algo sencilo y de utilidad de que se implemento es poder separar los números con _ con la finalidad que la lectura del número sea más fácil y clara.
  • ES13: se implementó el método at que recibe un valor numérico entero y devuelve el elemento en esa posición, permitiendo valores positivos y negativos. Los valores negativos contarán desde el último elemento del array.

¿Cómo se integra JavaScript a HTML?

Todas las formas de integrar JS a HTML se hace mediante la etiqueta script; esta se puede introducir tanto en head como en el body del documento.

Las formas de hacerlo son 3:

  • Mediante enlaces externos: apuntando el enlace a un archivo JS obtendremos como resultado un script en nuestro HTML. Este enlace debe ser colocado en el head y con el atributo src; por ejemplo: script src="#"script.
  • Mediante un script en el head: esto quiere decir que utilizaremos la etiqueta script para escribir código JS en el head del documento, ya que recordemos un script es un fragmento de código. Esto se hace cuando el script necesita estar disponible para la página mientras se está cargando.
  • Mediante un script en el body: es exactamente igual a la anterior forma, con la diferencia de escribir el script en el body del documento. Esto permite que el resto de la página se cargue sin tener que esperar a que se descarguen primero los recursos del JavaScript externo.
