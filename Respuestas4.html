¿Qué es el objeto dom en JS? Y ¿Cómo trabaja?

El Document Object Model o DOM permite a los programadores web acceder y manipular las páginas HTML como si fueran documentos XML.

El DOM trabaja mediante nodos dándole dinamismo a la página web; esto se da gracias a la transformación de los documentos, creando el DOM pudiendo darle la dinámica deseada. DOM transforma todos los documentos HTML en un conjunto de nodos que están interconectados que representan los contenidos de las páginas web y las relaciones entre ellos. La unión de todos los nodos se denomina árbol de nodos.

Por ejemplo en una estructura básica HTML con html, head, title, body y p; la raíz seria el documento HTML y a partir de acá cada nodo es de tipo element, realizando la conversión de forma jerárquica.

Existen muchos tipos de nodos, de los cuales más comunes son 5:

  • Document: nodo raíz del que derivan todos los demás nodos del árbol.
  • Element: representa cada una de las etiquetas HTML. Se trata del único nodo del que pueden derivar otros nodos.
  • Attr: define un nodo de este tipo para representar cada uno de los atributos de las etiquetas HTML.
  • Text: contiene el texto encerrado en una etiqueta.
  • Comment: se trata de los comentarios realizados en el HTML

¿Qué función tiene en JS el Object Map?

Su función primaria es la de almacenar pares claves-valor y que cualquier valor pueda ser utilizado tanto como clave o valor.

La propiedad más importante de este objeto es el size que permite conocer el tamaño del map. También cuenta con una serie de métodos, los cuales son:

  • • Get: devuelve el valor asociado a una clave, o undefined si no tiene ninguno.
  • • Set:establece un valor para la clave.
  • • Has: devuelve un valor booleano si existe un valor asociado a la clave en el map.
  • • Delete: elimina los valores asociados a una clave pasada por parámetro.
  • • Clear: elimina todos los elementos del objeto Map.

Y los siguientes iteradores:

  • Values: iterador con los valores para cada elemento de Map en orden de inserción.
  • Keys: iterador con las claves para cada elemento de Map en orden de inserción.
  • Entries: iterador con los pares clave-valor para cada elemento de Map en orden de inserción.
